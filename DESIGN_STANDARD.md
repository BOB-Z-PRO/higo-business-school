# HIGO 商学院设计标准（v1.1.1）

## 1. UI 定位
- 平台定位：高端商学院 + 生命科技研究院 + 国际化培训平台。
- 气质要求：简约大气、专业稳健、科技感克制、移动端 App 感。
- 禁止方向：SaaS 后台化冷淡风、低端营销页、模板化卡片堆砌。

## 2. 配色标准
- 深海军蓝：`#0B1F3A`（品牌主色、Hero、Footer）
- 高级藏蓝：`#102A43`（深色层级、标题强化）
- 暖白背景：`#F8F6F0`（页面主背景）
- 卡片白：`#FFFFFF`（模块容器）
- 高级金：`#C9A227`（主按钮与关键强调，小面积使用）
- 生命绿：`#2E8B57`（成长/完成状态）
- 科技青：`#00A6A6`（科技感点缀，小面积使用）
- 正文深色：`#1E293B`
- 正文灰色：`#64748B`

## 3. 字体与字号标准
- 字体：`Noto Sans SC` / 系统无衬线。
- H1：PC `40-48px`，移动端 `28-32px`。
- H2：PC `30-36px`，移动端 `22-26px`。
- 正文：PC `15-17px`，移动端 `14-15px`。
- 小字/标签：`11-13px`。
- 字距保持 `0`，不使用负字距。

## 4. 卡片设计标准
- 白底、浅边框、轻阴影、圆角 `16-18px`。
- 不使用重阴影，不做大面积高饱和色块。
- 图标小而清晰，避免巨大图形。
- 卡片标题明确，描述控制在 `2-3` 行优先。
- 模块区分优先使用：小标签、编号、细线，不靠整卡大色块。

## 5. 移动端布局标准（390px）
- 首页核心入口采用双列网格。
- 单卡高度适中，避免“一屏仅一张大卡”。
- 保证一屏可见 2 个以上有效入口。
- 禁止横向溢出，正文不出现超宽长行。
- 底部导航保持轻薄，不遮挡主内容。

## 6. 禁用风格清单
- 不使用 DNA / 细胞 / 粒子等复杂动态背景作为常规首页方案。
- 不使用大面积紫色渐变、荧光色和彩虹渐变。
- 不使用 Linear/Notion 式纯后台感视觉。
- 不使用保健品招商页式强刺激视觉。

## 7. 页面验收标准
- 首页主路径必须突出：
  1) 新人7天启动营
  2) 话术训练库
  3) 会议SOP库
  4) 合规表达替换库
- Hero 主按钮固定指向 `/business/survival/7-day`。
- 移动端 390px 无横向溢出，入口双列可读。
- 视觉统一、层次清晰，不出现风格混搭和模板感。
- 所有内容表达遵守合规边界：不出现收益确定性与医疗承诺表述。
